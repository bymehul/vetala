# AGENTS.md Update Strategy

Defines the strategy for updating an existing `AGENTS.md` file while preserving user-defined custom sections.

## Update Scope

Update mode refreshes **only** the standard sections generated by this skill. All other content in the file is treated as user-owned and preserved exactly as-is.

```yaml
- target: Standard sections (managed by skill)
  action: Re-analyze repository and replace with fresh content
- target: Custom sections (user-defined)
  action: Preserve unchanged, maintain original position
```

## Standard Sections (Managed)

### Single Repo / Package Document (5 Sections)

```yaml
- section: "1. Overview"
- section: "2. Folder Structure"
- section: "3. Core Behaviors & Patterns"
- section: "4. Conventions"
- section: "5. Working Agreements"
```

### Monorepo Root Document (3 Sections)

```yaml
- section: "1. Overview"
- section: "2. Folder Structure"
- section: "3. Working Agreements"
```

## Update Workflow

### Step 1: Read Existing AGENTS.md

Parse the existing file and identify all sections by their `##` headings.

### Step 2: Identify Standard Sections

Match each `##` heading against the standard section titles. Only matched sections are updated; all others are preserved as-is.

```yaml
- input: "## 1. Overview"
  matched: true
- input: "## 3. Core Behaviors & Patterns"
  matched: true
- input: "## 6. API Guidelines"
  matched: false (preserved as-is)
- input: "## Security Policy"
  matched: false (preserved as-is)
```

### Step 3: Re-analyze Repository

Perform the same repository analysis as initial generation:

- Measure LOC with `tokei` to determine character limits
- Analyze folder structure, patterns, conventions
- Compile updated content for each standard section

### Step 4: Reassemble Document

Reconstruct the document by:

1. Preserving the file title (`# AGENTS.md`) and any content before the first `##` heading
2. For each section in the original document order:
   - **Standard section** → Replace with newly generated content
   - **All other sections** → Keep original content unchanged
3. Non-standard sections interleaved between standard sections maintain their original position

```yaml
reassembly_rules:
  - rule: Preserve original section order
    detail: Do not reorder sections; update in place
  - rule: Standard sections get fresh content
    detail: Full replacement of section body under the heading
  - rule: Non-standard sections are untouched
    detail: Heading and all content preserved exactly
  - rule: Preserve leading content
    detail: Title and any preamble before first ## heading kept as-is
```

### Step 5: Write Updated File

Replace the existing `AGENTS.md` with the reassembled content.

## Section Matching Rules

Each `##` heading is compared against the exact standard section titles. Only headings that match exactly are managed; all others are preserved as-is.

```yaml
matching:
  strategy: exact_title
  standard_titles:
    single_repo:
      - "## 1. Overview"
      - "## 2. Folder Structure"
      - "## 3. Core Behaviors & Patterns"
      - "## 4. Conventions"
      - "## 5. Working Agreements"
    monorepo_root:
      - "## 1. Overview"
      - "## 2. Folder Structure"
      - "## 3. Working Agreements"
```

**Edge cases**:

```yaml
- case: Modified title (e.g., "## 1. Project Overview")
  action: No match; preserved as-is
- case: Standard section missing from existing file
  action: Insert at its defined position relative to other standard sections
- case: Duplicate section headings
  action: Only the first occurrence is matched as the standard section
```

## Monorepo Update Considerations

Update mode applies to each `AGENTS.md` file independently.

```yaml
- target: Root AGENTS.md
  managed_sections: 3 (Overview, Folder Structure, Working Agreements)
  custom_sections: All others preserved
- target: Package AGENTS.md
  managed_sections: 5 (Overview, Folder Structure, Core Behaviors & Patterns, Conventions, Working Agreements)
  custom_sections: All others preserved
```

**Document type transition**:

```yaml
- case: Single repo converted to monorepo (or vice versa)
  action: Force full regeneration instead of update
  reason: Section numbering differs between document types; update mode cannot reliably match sections across types
```

Update respects the same generation modes as initial generation:

```yaml
- mode: All
  behavior: Update root + all package AGENTS.md files
- mode: Root Only
  behavior: Update only root AGENTS.md
- mode: Single Package
  behavior: Update only the specified package AGENTS.md
```
